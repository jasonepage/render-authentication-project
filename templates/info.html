Auth Chat

A secure real-time chat platform built on FIDO2 WebAuthn. Authentication is restricted to physical security keys (no phone or platform authenticators) for a hardware-backed trust model.

Key Features

Hardware-bound Authentication

FIDO2 / WebAuthn with cross-platform attachment. Phone and platform authenticators are rejected.

Anonymous Identities

Users are assigned generated handles. Identities can be cycled without re-registering a key.

Public & Private Channels

Public read-only feed for anyone, gated send for authenticated users, and a private channel for verified members.

Minimal Data Footprint

No passwords, no email, no PII. Only the credential public key and a generated handle are stored.

Sharing Images

You can share images two ways:

1. @ prefix

@https://example.com/image.gif

2. /image command

/image https://example.com/image.png

Supported formats

  • GIF (.gif)
  • PNG (.png)
  • JPEG (.jpg, .jpeg)

Troubleshooting

  • Image not displaying: verify the URL ends with .gif, .png, .jpg, or .jpeg.
  • Image fails to send: confirm the URL is publicly accessible and the format is supported.
  • Security key not detected: WebAuthn requires HTTPS and a supported browser (Chrome, Firefox, Edge, Safari 14+).

Security

  • FIDO2 WebAuthn with cross-platform attachment only
  • Session cookies: Secure, HttpOnly, SameSite=Strict
  • Per-endpoint rate limiting
  • Image URL validation and output sanitization
  • No password storage
